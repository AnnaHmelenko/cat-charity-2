# QRKot — Charity Fund API

Backend-сервис для благотворительного фонда с автоматическим распределением пожертвований между проектами.

## Описание проекта

Проект решает задачу управления благотворительными сборами и распределения средств. Позволяет:

* создавать целевые проекты (сборы);
* принимать пожертвования от пользователей;
* автоматически распределять средства между проектами;
* управлять доступом через роли пользователей.

## Что реализовано

* REST API на FastAPI;
* асинхронная работа с БД (SQLAlchemy async);
* бизнес-логика:
    * автоматическое распределение пожертвований между проектами;
    * закрытие проектов при достижении целевой суммы;
    * обработка частичного инвестирования;
* ограничения:
    * запрет редактирования закрытых проектов;
    * контроль изменения суммы проекта;
    * запрет удаления проектов с инвестициями;
* аутентификация и управление пользователями (fastapi-users, JWT);
* ролевая модель (user / superuser);
* разграничение доступа к данным (разные представления для пользователей и администратора);
* миграции базы данных (Alembic);
* архитектура проекта (роутеры, сервисный слой, модели).

## Основные возможности

* регистрация и аутентификация пользователей;
* создание и управление благотворительными проектами;
* создание пожертвований;
* автоматическое распределение средств;
* просмотр собственных пожертвований;
* администрирование всех данных (для superuser);
* формирование Excel-отчётов по закрытым проектам с выгрузкой на Яндекс Диск.

## Технологический стек

* **Backend:** Python 3.12, FastAPI
* **База данных:** SQLite (dev)
* **ORM:** SQLAlchemy (async)
* **Миграции:** Alembic
* **Аутентификация:** fastapi-users (JWT)

## Установка и запуск

Клонирование репозитория:
git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
cd cat-charity-2

Виртуальное окружение:
python -m venv venv
venv\Scripts\activate     # Windows
source venv/bin/activate  # Linux / macOS

Установка зависимостей:
pip install -r requirements.txt

Миграции и запуск:
alembic upgrade head
uvicorn app.main:app --reload

После запуска документация API доступна по адресу http://127.0.0.1:8000/docs

## Примеры API

| Действие | Метод | Эндпоинт |
| --- | --- | --- |
| Регистрация пользователя | POST | `/auth/register` |
| Получение JWT | POST | `/auth/jwt/login` |
| Создание проекта (superuser) | POST | `/charity_project/` |
| Список проектов | GET | `/charity_project/` |

| Все пожертвования (superuser) | GET | `/donation/` |

## Отчёты в Яндекс Диск

Приложение умеет формировать Excel-отчёт по закрытым проектам (отсортированным по скорости сбора средств) и сохранять его на Яндекс Диск, возвращая публичную ссылку.

### Настройка

Для работы функции добавьте в файл `.env` токен Яндекс Диска:

YANDEX_DISK_TOKEN=ваш_токен

Токен можно получить на [полигоне Яндекс Диска](https://yandex.ru/dev/disk/poligon/).

Опционально можно задать формат даты в имени файла отчёта:
REPORT_FORMAT=%d.%m.%Y_%H-%M-%S

### Использование

Эндпоинт доступен только суперпользователю:

| Действие | Метод | Эндпоинт |
| --- | --- | --- |
| Сформировать отчёт | POST | `/yandex/` |

В ответе возвращается публичная ссылка на созданный Excel-файл. Файл содержит таблицу с названиями проектов, временем сбора и описанием, а также итоговую строку с общим количеством проектов. Если закрытых проектов нет, эндпоинт возвращает ошибку 404.
